<objective>
## Goal
Close Milestone 20 by adding a repo-owned **GitHub repository rename and migration proof checklist**, drift guards that keep the public docs aligned, and a blocking human-action checkpoint for the actual GitHub rename decision. The plan prepares and verifies the `coctostan/pi-ptc-advanced` repository identity without allowing automated APPLY to perform the external rename.

## Purpose
Phases 58–60 made the package/release surface clean for `pi-ptc-advanced@1.0.0` and documented manual publish boundaries. Phase 61 completes the remaining public-identity transition: maintainers need a deterministic path for the GitHub repo rename, old-name redirect expectations, local remote migration, CI/package metadata verification after rename, and clear docs that do not falsely claim the rename happened before the user performs it.

## Output
- Extended `test/release-readiness.test.ts` drift guards for repository-rename migration docs and active-doc cross-links.
- New `docs/releases/REPO-RENAME-CHECKLIST.md` with pre-rename, human-action rename, redirect/remote migration, post-rename verification, rollback/deferral notes, and explicit automated-APPLY stop boundary.
- Updated `docs/releases/PUBLISH-CHECKLIST.md`, `docs/releases/1.0.0.md`, `docs/personal-fork-maintenance.md`, and `README.md` to point to the rename checklist while preserving accurate "intended/manual" wording until the user confirms the rename.
- Blocking checkpoint for the user to either perform the GitHub rename manually or explicitly defer it; APPLY records the outcome for UNIFY.
</objective>
