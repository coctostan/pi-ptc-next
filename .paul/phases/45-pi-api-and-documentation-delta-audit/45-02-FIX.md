## Fix
Unblock the Phase 45 merge gate by upgrading the GitHub Actions runner
Node version so `node --test test/*.test.ts` works.

CI on PR #1 is failing with
`TypeError [ERR_UNKNOWN_FILE_EXTENSION]: Unknown file extension ".ts" for .../test/utils.test.ts`
on Node 20.20.2. The repository's test command (`npm run verify:personal`
and `npm test` both call `node --test test/*.test.ts` directly) relies
on Node's native TypeScript loader, which is available since Node 22.6+
and stable from Node 22 LTS onward. Local development is on Node 25.9.0,
where this works. The fix is to bump `.github/workflows/ci.yml` from
`node-version: 20` to `node-version: 22` to match the local execution
environment.

This is a CI-only fix: no `src/**`, `test/**`, `package.json`, or
runtime code is changed. It is a side-loop fix to make PR #1 mergeable
and to unblock Phase 46 (`require_pr_before_next_phase: true`).
</objective>
