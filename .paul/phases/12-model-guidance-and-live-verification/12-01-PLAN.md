<objective>
## Goal
Update the model-facing `code_execution` tool description so the helper signatures and examples accurately reflect the normalized return types after Phase 11, and verify all helpers work end-to-end in a live session.

## Purpose
The helper signatures in the `code_execution` description still say `ptc.read_many -> list[Union[str, ReadResult]]`. After Phase 11, it always returns `list[str]`. Models generating code from stale signatures may write unnecessary type-checking boilerplate or miss that `ptc.read_text()` is now the reliable string accessor.

## Output
- Updated helper signatures in the code_execution tool description
- Clarified guidance on when to use `read()` (structured) vs `ptc.read_text()` (string)
- Live verification that all helpers return their documented types
</objective>
