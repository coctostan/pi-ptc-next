<objective>
## Goal
Modify the `pi-hashline-readmap` extension so it emits its tool executor functions on pi's EventBus after registration, making them discoverable by other extensions (specifically `pi-ptc-next`).

## Purpose
Pi's `getAllTools()` API intentionally strips `execute` functions, so `pi-ptc-next` cannot reach hashline's tool executors for nested PTC tool calls. This plan implements the emitter half of a temporary EventBus bridge that will be removed when pi adds `getToolExecutor()` to the ExtensionAPI.

## Output
- 4 `register*Tool()` functions return their tool definitions after registering
- `index.ts` emits all 4 tool executors on `"hashline:tool-executors"` channel and stashes on `globalThis` for load-order safety
- ~16 lines changed across 5 files in the hashline repo
</objective>
