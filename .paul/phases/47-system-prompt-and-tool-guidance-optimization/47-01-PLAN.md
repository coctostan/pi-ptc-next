<objective>
## Goal
Add Pi prompt metadata support for `code_execution` and custom tools so PTC guidance appears in the default system prompt through `promptSnippet`, `promptGuidelines`, and idempotent `before_agent_start` routing.

## Purpose
Phase 45 found that `code_execution` is omitted from Pi's default "Available tools" prompt section and that custom-tool prompt metadata is dropped at registration. Phase 46 aligned the runtime to the latest Mario-scope Pi baseline; this plan uses the prompt metadata APIs available in that baseline without claiming an `@earendil-works/*` migration.

## Output
- `code_execution` registered with a concise `promptSnippet` and active-only `promptGuidelines`.
- A human-reviewed prompt draft is proposed, edited collaboratively, and approved before implementation.
- Auto-route prompt injection made idempotent with `systemPromptOptions` and existing prompt/guideline checks.
- Custom tools preserve user-authored `promptSnippet` / `promptGuidelines` through `CustomToolManager` registration.
- Focused regression tests plus README/CHANGELOG documentation for the new prompt integration behavior.
</objective>
