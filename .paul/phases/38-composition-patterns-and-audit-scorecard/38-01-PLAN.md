    Create `test/live-audit-composition.test.ts` using the same CodeExecutor + stub-tool harness. Define stub tools that support multi-step workflows:
    - `read` — returns file content
    - `grep` — returns structured matches with anchors
    - `search_tool` — returns structured results with ptcValue
    - `detail_tool` — returns detailed info for a given ID

    **Composition workflow tests (~6-8 tests):**

    1. **Search → Inspect → Summarize:** grep for a pattern, read the matched files, fit_output the combined result. Assert bounded output with search evidence.

    2. **Batch read → Reduce → Fit:** read_many 4 files, reduce_tool to accumulate line counts, fit_output the summary. Assert final output is compact JSON.

    3. **Introspection-gated branching:** list_callable_tools, check if "detail_tool" is available, branch on result. Assert correct branch taken.

    4. **First-success fallback chain:** Try search_tool (fails), then grep (succeeds), then fit_output the result. Assert fallback produces correct output.

    5. **Handle extraction workflow:** Call search_tool (returns ptcValue with responseId), extract_handles, use first_handle to get the response handle. Assert handle extracted.

    6. **Error-resilient pipeline:** batch_tool with mix of succeeding/failing calls wrapped in try/except, gather surviving results, fit_output. Assert partial results captured.

    7. **Full recipe-style workflow:** Combine batch_tool → gather_limit → reduce_tool → fit_output in a single execution. Assert the entire chain produces bounded compact output.

    Each test should execute multi-line Python through the CodeExecutor, chaining 2-4 helpers per test. Keep the file ~150-250 lines.
