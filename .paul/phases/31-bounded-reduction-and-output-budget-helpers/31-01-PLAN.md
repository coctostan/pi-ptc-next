<objective>
## Goal
Add one bounded reducer helper and one bounded output-budget helper to the Python runtime so model-written `code_execution` scripts can collapse large intermediate tool results into compact final outcomes without re-implementing the same reduction and truncation plumbing in user code.

## Purpose
Phase 31 should build directly on Phase 30's orchestration primitives while preserving the repo's narrow-helper philosophy. The new helpers should make it easier to keep large intermediate results local to Python, return compact JSON-safe summaries, and stay aligned with the existing `code_execution` output cap without expanding into broader concurrency APIs or docs work.

## Output
- Session output-budget plumbing available inside the Python runtime
- `ptc.reduce_tool(...)` in the Python runtime
- `ptc.fit_output(...)` in the Python runtime
- Focused execution-level proof for bounded reduction and output-budget behavior
</objective>
