<objective>
## Goal
Make Phase 5 concrete by upgrading the Python helper contract surface so hashline-native `read` / `grep` / `edit` results are described with clearer structured types and model-facing guidance instead of only generic fallback signatures.

## Purpose
Milestones 1–2 proved correctness, passthrough, and lightweight interop. This phase improves usability: extension authors and models should be able to understand the structured return contract for anchored data without reverse-engineering raw JSON blobs or assuming fallback helper signatures are the only supported shape.

## Output
- Richer helper-contract/type mapping for structured hashline-style builtin results
- Updated generated Python wrapper models/signatures for those helper contracts
- Regression coverage and `code_execution` guidance updates reflecting the improved ergonomics
</objective>
