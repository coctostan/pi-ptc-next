<objective>
## Goal
Introduce `ptc.report(...)` as a recognized, JSON-safe report shape for `code_execution` returns, and render that shape through Pi's existing completed tool-result UI while keeping free-form returns unchanged.

## Purpose
Milestone 18 is about turning repo/dataset analysis into a small, well-shaped, debuggable answer. Phase 49 made completed code debuggable through expansion. Phase 50 adds the first explicit output shape: a soft contract that lets Python return title, scalar metrics, ranked/table rows, samples, and warnings in a predictable structure that tests and future evals can assert against. The contract must stay optional: existing strings, dicts, lists, and `ptc.fit_output(...)` results continue to work exactly as before.

## Output
- A repo-owned TypeScript report contract (`src/report.ts`) plus public type export.
- A Python helper `ptc.report(...)` that builds the recognized report shape and validates obvious bad inputs with clear errors.
- RPC/details plumbing so completed `code_execution` results expose `details.report` and `details.reportProduced` for recognized reports.
- Completed-result rendering that shows compact report output by default and fuller report details when expanded.
- Focused runtime/render/contract tests plus README and CHANGELOG updates.
</objective>
