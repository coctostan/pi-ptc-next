<objective>
## Goal
Fix `ptc.read_text()`, `ptc.read_many()`, and `ptc.read_tree()` so they return strings as documented, normalizing structured `ReadResult` dicts back to raw text when the hashline bridge is active.

## Purpose
The Milestone 4 bridge made `read()` return structured `ReadResult` dicts instead of strings. The `ptc.*` helpers that wrap `read()` promise string returns but now pass through dicts, breaking common agent patterns like `content.split("\n")` and `content.count("\n")`.

## Output
- `_extract_text()` helper function in runtime.py
- `read_text()` always returns `str`
- `read_many()` always returns `list[str]`
- `read_tree()` returns `list[dict]` with `content` as `str`
- `read()` itself unchanged — still `Union[str, ReadResult]`
</objective>

  <action>
    Add a module-level helper function `_extract_text()` that converts a ReadResult dict to a plain string, and apply it in the three affected helpers.

    1. Add `_extract_text()` before the `_PtcHelpers` class (around line 73):

    ```python
    def _extract_text(result: Any) -> str:
        """Extract raw text from a ReadResult dict, or return as-is if already a string."""
        if isinstance(result, str):
            return result
        if isinstance(result, dict) and "lines" in result:
            return "\n".join(line.get("raw", "") for line in result["lines"])
        return str(result)
    ```

    2. Update `read_text()` (line 96–97):
    ```python
    async def read_text(self, path: str, offset: int | None = None, limit: int | None = None) -> str:
        result = await read(path=path, offset=offset, limit=limit)
        return _extract_text(result)
    ```

    3. Update `read_many()` (line 99–110) — normalize each result:
    ```python
    async def read_many(
        self,
        paths: Sequence[str],
        max_concurrency: int | None = None,
        *,
        offset: int | None = None,
        line_limit: int | None = None,
    ) -> Sequence[str]:
        results = await self.gather_limit(
            [read(path=path, offset=offset, limit=line_limit) for path in paths],
            limit=max_concurrency,
        )
        return [_extract_text(r) for r in results]
    ```

    4. Update `read_tree()` (line 112–129) — normalize the content field:
    ```python
    async def read_tree(
        self,
        pattern: str,
        path: str = ".",
        max_files: int = 1000,
        concurrency: int | None = None,
        offset: int | None = None,
        line_limit: int | None = None,
    ) -> Sequence[dict[str, Any]]:
        files = await self.find_files_abs(pattern=pattern, path=path, max_files=max_files)
        contents = await self.read_many(files, max_concurrency=concurrency, offset=offset, line_limit=line_limit)
        return [
            {
                "path": file_path,
                "content": content,
            }
            for file_path, content in zip(files, contents)
        ]
    ```

    Note: `read_tree` already calls `read_many`, which now normalizes. So `read_tree` needs no additional change beyond what `read_many` provides.

    Avoid: Changing `read()` itself. It must remain `Union[str, ReadResult]`.
    Avoid: Changing any other runtime.py behavior (stdout proxy, RPC, tracing).
