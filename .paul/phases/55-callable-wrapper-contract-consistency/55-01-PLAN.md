<objective>
## Goal
Make the Python callable-tool wrapper contract consistent and test-backed for `read`, `grep`, `find`, `glob`, and `ls`: all callable Pi wrappers are documented as awaitable, common first positional shorthand is consistent where supported, and generated guidance no longer implies synchronous direct wrappers.

## Purpose
Milestone 19 is converting live `code_execution` helper findings into harder runtime behavior and clearer contracts. Phase 55 closes the most user-visible callable-wrapper ambiguity without reopening Phase 56 result-shape/partial-error semantics or Phase 54 runner availability.

## Output
- Focused TDD coverage for callable wrapper descriptions, generated wrapper code, live-audit helper usage, and generated `code_execution` guidance.
- Runtime wrapper-generation changes that preserve async RPC execution while normalizing obvious first positional shorthand, especially `grep("pattern", path=...)`.
- README and generated tool-description updates that clearly distinguish awaitable callable Pi wrappers from `ptc.*` helpers.
</objective>
