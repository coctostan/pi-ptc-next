<objective>
## Goal
Tighten `ptc.run_tests(pattern)` so its live report output makes runner availability and command rendering unambiguous without widening the helper beyond Node `node --test`.

## Purpose
Phase 53 intentionally made missing test runners structured data, but live `code_execution` testing showed the default Docker runtime can surface only `runner_available=false` reports and the current command string is ambiguous for patterns containing spaces. This plan keeps the helper safe and narrow while making the unavailable state and command metadata more actionable.

## Output
- Focused tests proving runner path/resolution metadata, improved unavailable warnings, and command display behavior.
- Runtime changes in `src/python-runtime/runtime.py` preserving shell-free execution while adding scalar report metadata.
- Prompt/README guidance explaining that `ptc.run_tests` depends on Node availability in the active runtime and may report runner-unavailable in Docker/Python-only environments.
</objective>
