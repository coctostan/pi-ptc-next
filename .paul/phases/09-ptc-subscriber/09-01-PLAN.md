<objective>
## Goal
Add EventBus subscriber and `globalThis` pre-emit check to `ToolRegistry` so that hashline tool executors are overlaid onto builtins in `buildToolMap()`, routing nested PTC tool calls through the real hashline `read`/`grep`/`edit`/`sg` implementations.

## Purpose
Phase 8 confirmed the hashline extension emits its tool executors on `"hashline:tool-executors"` and stashes them on `globalThis.__hashlineToolExecutors`. This phase makes PTC consume that payload so nested `code_execution` tool calls deliver structured `ptcValue` payloads to Python.

## Output
- `ToolRegistry` gains an `extensionExecutors` map, populated from EventBus + `globalThis`
- `buildToolMap()` overlays extension executors onto builtins before the `pi.getAllTools()` loop
- Unit + integration tests verify overlay behavior, callable resolution, and fallback
</objective>

    ```typescript
    constructor(private pi: ExtensionAPI) {
      // Bridge: consume hashline tool executors from EventBus.
      // Remove when pi exposes getToolExecutor() on ExtensionAPI.
      // Grep marker: "hashline:tool-executors"
      const preEmitted = (globalThis as any).__hashlineToolExecutors;
      if (preEmitted) {
        this.ingestExtensionExecutors(preEmitted);
      }
      pi.events.on("hashline:tool-executors", (data: unknown) => {
        this.ingestExtensionExecutors(data);
      });
    }
    ```

    3. Add a private method to process the payload:
    ```typescript
    private ingestExtensionExecutors(data: unknown): void {
      if (!data || typeof data !== "object") return;
      for (const [, toolDef] of Object.entries(data as Record<string, unknown>)) {
        if (!toolDef || typeof toolDef !== "object") continue;
        const t = toolDef as { name?: string; execute?: unknown; ptc?: PtcToolOptions; parameters?: unknown };
        if (typeof t.name !== "string" || typeof t.execute !== "function") continue;
        const classification = classifyTool(t.name, t.ptc);
        this.extensionExecutors.set(t.name, {
          name: t.name,
          description: "",
          parameters: t.parameters ?? { type: "object", properties: {} },
          execute: t.execute as InternalToolExecute,
          ptc: classification.ptc,
          source: "extension",
          isReadOnly: classification.isReadOnly,
        });
      }
    }
    ```

    4. In `buildToolMap()`, after the builtin loop (line ~173) and before the customTools loop (line ~175), add the overlay block:
    ```typescript
    // Bridge: overlay extension executors from EventBus.
    // Remove when pi exposes getToolExecutor() on ExtensionAPI.
    for (const [name, extTool] of this.extensionExecutors) {
      const existing = allTools.get(name);
      if (existing) {
        allTools.set(name, {
          ...existing,
          execute: extTool.execute,
          ptc: extTool.ptc ?? existing.ptc,
          isReadOnly: extTool.isReadOnly,
          source: "extension",
        });
      } else {
        allTools.set(name, extTool);
      }
    }
    ```

    Avoid:
    - Changing existing builtin creation, customTool, or pi.getAllTools() logic
    - Modifying any execute function signatures or tool policy filtering
    - Adding the overlay after the pi.getAllTools() loop (it must come before, so metadata from pi.getAllTools() can still enrich the entry)
