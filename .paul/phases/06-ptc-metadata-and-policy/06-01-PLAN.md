<objective>
## Goal
Clarify and extend the PTC metadata/policy contract for extension tools so callable-tool exposure is driven by explicit, documented annotations instead of only the current `enabled` / `readOnly` pair and environment allowlists.

## Purpose
Milestones 1–5 proved runtime correctness and improved Python helper ergonomics, but Phase 6 still needs a cleaner contract for extension authors and operators. `pi-ptc-next` should make it obvious which tools are callable from Python, which are safe under read-only policy, and how those decisions are configured when combining PTC with hashline-native tools.

## Output
- Expanded `ptc.*` metadata contract for extension tools and settings
- Runtime/policy enforcement updates that honor the clarified metadata without regressing existing builtin behavior
- Regression coverage plus hashline-integration docs describing recommended metadata and policy setup
</objective>
