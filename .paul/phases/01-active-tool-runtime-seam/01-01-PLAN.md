<objective>
## Goal
Make `code_execution` invoke the same active Pi tool executor that chat uses for same-name overridden builtins such as `read`, `grep`, and `edit`, while preserving builtin fallback behavior and current policy gates.

## Purpose
This removes split-brain tool behavior in Milestone 1 so hashline-native tool overrides behave consistently inside Python workflows, which is the trust foundation for later structured `details.ptcValue` interop.

## Output
- Runtime resolution update in `src/tool-registry.ts`
- Any strictly necessary supporting type adjustment in `src/contracts/tool-types.ts`
- Focused regression coverage for override, fallback, policy, and recursion behavior
- Minimal milestone doc note describing the new resolution behavior
</objective>
