<objective>
## Goal
Add small, test-driven Python helper ergonomics for `code_execution`: workspace-root-aware path formatting options on file discovery/tree helpers plus slim `ptc.tabulate(...)` and `ptc.diff(...)` bridge helpers.

## Purpose
Phase 51 makes common repo/dataset analysis scripts easier to write and easier to return as Phase 50 reports without adding broad semantic data helpers that `nu` already covers. The slice must preserve existing helper defaults and free-form return compatibility.

## Output
- `src/python-runtime/runtime.py` helper updates for:
  - `ptc.find_files(..., relative=True, relative_to=None)` with current relative default preserved.
  - `ptc.find_files_abs(..., relative=False, relative_to=None)` with current absolute default preserved.
  - `ptc.read_tree(..., relative=False, relative_to=None)` so callers can opt into relative entry paths while reads still use absolute/host-safe paths internally.
  - `ptc.tabulate(rows, headers=None, title=None)` returning a Phase-50-compatible report table object.
  - `ptc.diff(before, after)` returning a deterministic JSON-safe structural diff for dicts and list/set-like sequences.
- Focused runtime/helper tests and README/tool-description/CHANGELOG updates.
</objective>
