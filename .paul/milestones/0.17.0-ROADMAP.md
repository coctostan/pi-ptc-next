## Phase 49: Pi TUI Audit and Collapsible Code Body

Focus: Audit Pi's TUI primitives and replace completed `code_execution` source loss with an expanded completed-result view that preserves the Python source after a call completes, while adding prompt guidance for when to prefer `nu` versus `code_execution`.

Outcome: Recorded the Pi TUI audit artifact, added completed `code_execution` collapsed/expanded Python-source rendering, clarified `nu` vs `code_execution` prompt guidance, added focused render/prompt tests, updated README/CHANGELOG, reconciled results in `49-01-SUMMARY.md`, and merged PR #5.

## Phase 50: Structured Report Type

Focus: Introduce `ptc.report(...)` as an optional soft contract for `code_execution` returns with title, scalar metrics, tables, samples, and warnings, while preserving free-form returns.

Outcome: Added the canonical `ptc_report` helper, preserved `details.report` / `details.reportProduced`, added compact/expanded report rendering, updated docs/contract tests, completed full verification, reconciled results in `50-01-SUMMARY.md`, and merged PR #6.

## Phase 51: Path Ergonomics and Bridge Helpers

Focus: Add root-aware path formatting to file helpers and add slim bridge helpers (`ptc.tabulate(...)`, shallow `ptc.diff(...)`) without adding broad aggregators already covered by `nu`.

Outcome: Added `relative` / `relative_to` options on `ptc.find_files(...)`, `ptc.find_files_abs(...)`, and `ptc.read_tree(...)`; added report-compatible `ptc.tabulate(...)` and shallow `ptc.diff(...)`; updated docs/prompt guidance; reconciled results in `51-01-SUMMARY.md`; and merged PR #7.

## Phase 52: Callable-Tool Introspection

Focus: Propagate optional callable-tool prompt metadata to Python and add targeted on-demand runtime help without inflating the generated `code_execution` tool description.

Outcome: Added optional `promptSnippet` / `promptGuidelines` to callable-tool metadata, implemented `ptc.help(tool_name)` while preserving schema-only `ptc.get_tool_schema(...)`, updated README/generated guidance/CHANGELOG and focused tests, reconciled results in `52-01-SUMMARY.md`, and merged PR #8.

## Phase 53: Test Runner Verb

Focus: Add `ptc.run_tests(pattern)` as a first-class helper that runs Node's built-in `node --test` and returns a structured Phase 50 report with pass/fail/duration metrics and bounded failure details.

Outcome: Added `ptc.run_tests(pattern)` with safe argv subprocess execution, invalid-pattern validation, runner-unavailable and timeout-as-data behavior, reentry-safe env scrubbing, README/CHANGELOG/generated-guidance coverage, focused live tests, full verification at 236 passing / 0 failing, and merged PR #9.
