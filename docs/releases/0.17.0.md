# Release Notes — pi-ptc-advanced 0.17.0

Date: 2026-05-12

## Summary

`0.17.0` closes Milestone 18 — PTC Leverage and Output Shape. The release makes `code_execution` more useful for repo and dataset analysis by improving completed-result debugging, adding a structured report contract, adding small bridge helpers for path/table/diff work, exposing targeted callable-tool help, and introducing a Node `node --test` runner helper.

## Added

- Completed `code_execution` results keep default output compact while exposing executed Python source in expanded tool details.
- `ptc.report(...)` provides an optional JSON-safe structured report shape with title, metrics, tables, samples, and warnings; recognized reports are preserved in `details.report` / `details.reportProduced`.
- `ptc.find_files(...)`, `ptc.find_files_abs(...)`, and `ptc.read_tree(...)` support root-aware `relative` / `relative_to` formatting options while preserving historical defaults.
- `ptc.tabulate(...)` returns report-compatible table payloads, and shallow `ptc.diff(...)` captures explicit before/after changes without becoming a recursive diff engine.
- Callable-tool metadata can carry optional prompt guidance into Python, and `ptc.help(tool_name)` returns bounded on-demand metadata including parameters.
- `ptc.run_tests(pattern)` runs Node's built-in `node --test` from the active runtime workspace and returns a `ptc_report` with pass/fail/duration metrics, runner availability, a bounded failures table, and a fixed 120s timeout.

## Changed

- Package metadata and release-package verification now target `pi-ptc-advanced@0.17.0`.
- README, CHANGELOG, generated `code_execution` guidance, and focused contract tests now describe the Milestone 18 helper/report surface.
- The prompt guidance keeps broad grouping, ranking, and histogram work in `nu` while steering custom/stateful multi-tool orchestration to `code_execution`.

## Verification

- Final Phase 53 verification: `npm test` — 236 passing / 0 failing.
- Final Phase 53 verification: `npm run build` — clean.
- Dependency audit baseline remained unchanged at `0 critical / 0 high / 3 moderate` through Milestone 18.

## Deferred / not included

- Cross-runner support for vitest, jest, pytest, or package-script dispatch is intentionally out of scope for `ptc.run_tests(pattern)`.
- Caching, cross-call session state, helper persistence, and broad PTC aggregator helpers remain deferred.
- Automated npm publish remains manual; the repo-local verification commands validate the release surface but do not publish to a registry.
