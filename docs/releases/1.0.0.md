# pi-ptc-advanced 1.0.0

`pi-ptc-advanced@1.0.0` is the public identity baseline for this package. It updates the active package and release-surface metadata from the earlier fork-first `pi-ptc-next` framing to the intended public `pi-ptc-advanced` package and `coctostan/pi-ptc-advanced` repository identity, and polishes the user-facing README/runbook documentation to read like a public package front door instead of a fork-history note.

## Identity baseline

- Package name: `pi-ptc-advanced`
- Version: `1.0.0`
- Intended public repository identity: `coctostan/pi-ptc-advanced`
- Prior fork/repo lineage remains credited in [`README.md`](../../README.md) and historical release material.

## What is in this release

- README rewritten into a public package front door with Quick Start, Installation, Usage, Python helpers, Configuration, Verification, Limitations, Development, Troubleshooting, License, and Credits/lineage sections.
- Public release maintenance runbook ([`../personal-fork-maintenance.md`](../personal-fork-maintenance.md)) reframed to lead with public release maintenance instead of personal-fork-first wording, while preserving the explicit manual git sync/upgrade boundary.
- Active release docs point to this `1.0.0` note; older release notes remain historical context.
- Release-readiness drift tests extended to guard the Phase 59 documentation contract (section structure, no premature npm registry install commands, no stale "personal fork" framing, no implied publish/tag/release/rename claims).
- Package metadata, package-lock root metadata, and `scripts/verify-release-package.sh` continue to target `pi-ptc-advanced@1.0.0`.

## Verification surface

This baseline is guarded by:

- `node --test test/release-readiness.test.ts`
- `npm run build`
- `bash scripts/verify-release-package.sh`
- `npm run verify:ci`

The release-package verifier checks package metadata, the `npm pack --dry-run` tarball surface, and a clean install from the packed tarball into a temporary directory for `pi-ptc-advanced@1.0.0`. GitHub Actions at [`.github/workflows/ci.yml`](../../.github/workflows/ci.yml) calls the same repo-owned `verify:ci` command so local and CI verification stay aligned.

For day-to-day maintainer verification, see the public release maintenance runbook at [`../personal-fork-maintenance.md`](../personal-fork-maintenance.md).

## Manual boundaries

This release note does not claim that publishing has happened. Actual `npm publish`, npm dist-tags, git tags, GitHub releases, and the GitHub repository rename to `coctostan/pi-ptc-advanced` are manual/user-owned operations and remain outside automated APPLY work. Until publish is confirmed, README/runbook/release docs continue to install from Git source rather than the npm registry.

See [PUBLISH-CHECKLIST.md](./PUBLISH-CHECKLIST.md) for the dry-run + manual publish sequence that pairs with this release.
